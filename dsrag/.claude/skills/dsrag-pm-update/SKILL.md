# DSRAG PM Update

Generate a project management status report from consolidated knowledge fragments.

## When to Use

- Regular status reporting to stakeholders
- Before steering committee meetings
- When asked "what's the project status?"
- After consolidation to summarize project health

## Prerequisites

- Project must be initialized with `dsrag-init-project`
- Consolidation must have run (`/dsrag-consolidate`)
- At least one consolidation fragment must exist

## Usage

```bash
/dsrag-pm-update [project-id]
```

**Parameters:**
- `project_id` (required): Project identifier

## Output

```
[project-id]/consolidation/pm_status_report.md
```

Overwrites on each run — git provides version history.

---

## Execution Process

### Step 1: Validate Prerequisites

Check:
- Project exists at `.dsrag/[project_id]/`
- Consolidation directory exists at `[project-id]/consolidation/`
- At least one fragment file exists

If consolidation not found:
```markdown
⚠️ Consolidation required before generating PM status report.

Run `/dsrag-consolidate [project-id]` first, then re-run this skill.

Would you like to run consolidation now?
```

If user says yes, invoke `/dsrag-consolidate [project-id]` then continue.

### Step 2: Read Consolidation Fragments

Read these fragments (all optional — use what exists):

| Fragment | Status Report Section |
|----------|----------------------|
| `roadmap.md` | What was done, What's next |
| `problems.md` | Current blockers |
| `risks.md` | Dependencies, Risk status |
| `stakeholders.md` | Key stakeholder status |
| `requirements.md` | Requirements completion |
| `processes.md` | Process maturity |
| `capabilities.md` | Capability gaps |
| `financial.md` | Budget/cost status |
| `index.md` | Completeness scores |

## Design Decisions

| Decision | Rationale |
|----------|-----------|
| Reads from consolidation, not raw knowledge | Consolidation is already synthesized and structured — more reliable for status |
| Overwrites on each run | Status report is point-in-time; git provides history |
| Auto-determines RAG status | Based on blockers, dependencies, and roadmap — not manually set |
| Offers to run consolidation if missing | Reduces friction — user doesn't have to remember prerequisites |
