# DSRAG Public Repository — Product Requirements

## Vision

Provide a community hub where users can learn about, install, and contribute to DSRAG.

## Goals

1. **User onboarding** — Clear documentation so anyone with Claude Code CLI can get DSRAG running
2. **Contributor onboarding** — Format specs, examples, and guides so contributors can add lenses and templates
3. **System distribution** — The installable DSRAG system available for users to copy into their projects

## Audiences

### Users
Consultants and analysts who want to use DSRAG to extract structured knowledge from interviews and documents. They need: getting started guide, feature documentation, skill reference.

### Contributors
Developers who want to extend DSRAG with new extraction lenses, delivery templates, or improvements. They need: content guide, extension guide, contribution workflow, example test data.

## What This Repository Contains

| Component | Location | Purpose |
|-----------|----------|---------|
| Community docs | `docs/` | Learning, onboarding, reference |
| Installable system | `dsrag/` | Skills, agents, scripts to copy into projects |
| Contribution guides | `CONTRIBUTING.md`, `docs/content-guide.md` | How to contribute |
| Issue templates | `.github/ISSUE_TEMPLATE/` | Structured contribution proposals |

## What This Repository Does NOT Contain

- Internal development plans and design documents
- Internal reference documentation
- Development-specific Claude Code configuration
- Project-specific data or knowledge bases
